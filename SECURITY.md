# Security Policy

## Supported Surface

Security-sensitive areas in this repository include:

- wallet authentication and session handling
- task mutation routes
- transaction verification and escrow reconciliation
- proof submission and signer operations
- Supabase access, service-role use, and RLS policy bootstrap

## Reporting A Vulnerability

Do not open public issues for security reports.

Use GitHub private vulnerability reporting for this repository if it is enabled. If it is not enabled, contact the maintainers through a private channel before disclosing details publicly.

Include:

- affected route, page, or contract
- reproduction steps
- impact
- any proof-of-concept artifacts

## Response Expectations

- acknowledge receipt
- validate impact
- contain or mitigate if needed
- ship a fix
- publish follow-up guidance if public action is required

There is no bug bounty or SLA declared in this repository today.
